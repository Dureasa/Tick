# Tick 需求文档

## 概述

Tick 是一款运行在终端内的待办事项管理工具（TUI），核心理念是简洁高效、类Vim交互、美观实用。目标用户为个人使用，不涉及多用户、同步、商业场景。

---

## 技术选型

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 开发语言 | Python | 生态丰富、开发效率高 |
| TUI框架 | textual | 现代化、组件丰富、异步支持 |
| 数据格式 | JSON | 简单直接、可读性好、易于手动编辑 |
| 存储位置 | `.tick/` 目录 | 本地存储、轻量无负担 |

---

## 数据模型

### 任务 (Task)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，UUID格式 |
| title | string | 是 | 任务标题 |
| status | enum | 是 | 状态：`pending` / `completed` |
| priority | enum | 否 | 优先级：`high` / `medium` / `low` |
| due_date | date | 否 | 截止日期，格式 `YYYY-MM-DD` |
| category | string | 否 | 分类，自由输入 |
| note | string | 否 | 备注详情 |
| created_at | datetime | 是 | 创建时间 |
| completed_at | datetime | 否 | 完成时间 |

### 优先级颜色

| 优先级 | 颜色 |
|--------|------|
| high | 红色 |
| medium | 黄色 |
| low | 绿色 |

## 功能需求

### 1. 主界面布局

```
┌─────────────────────────────────────────────────┐
│  Tick                                    [视图名]│
├─────────────────────────────────────────────────┤
│                                                 │
│  任务列表区域                                    │
│                                                 │
├─────────────────────────────────────────────────┤
│  状态栏: 总数 | 已完成 | 当前视图 | 快捷键提示    │
└─────────────────────────────────────────────────┘
```

### 2. 视图切换

| 快捷键 | 视图 | 说明 |
|--------|------|------|
| 1 | 清单视图 | 默认视图，任务列表形式 |
| 2 | 日历视图 | 月历形式，每天显示任务数量 |
| 3 | 分类视图 | 按分类分组展示任务 |

### 3. 清单视图

**任务显示规则**：
- 未完成任务：正常显示
- 已完成任务：当天显示，第二天之后隐藏
- 过期任务：高亮显示（截止日期 < 今天）

**排序规则**（不可手动排序）：
1. 按截止日期升序（无截止日期的排在最后）
2. 同一天按优先级排序：high > medium > low > 无优先级
3. 同优先级按创建时间排序

### 4. 日历视图

- 显示当前月份的月历网格
- 每个日期格子显示任务数量（如 "3" 表示当天有3个任务）
- 有任务的日期可以点击/回车查看详情
- 过期未完成的日期高亮

### 5. 分类视图

- 按分类分组显示任务
- 无分类的任务归入"未分类"组
- 每个分类显示任务数量

### 6. 任务操作

| 操作 | 快捷键（默认） | 说明 |
|------|----------------|------|
| 新建任务 | n | 弹出表单填写各字段 |
| 编辑任务 | e | 分字段编辑当前任务 |
| 删除任务 | d | 直接删除（需确认） |
| 完成切换 | Enter | 切换任务完成状态 |
| 上下导航 | j/k 或 ↑/↓ | 选择任务 |
| 退出 | q | 退出应用 |

> 注：快捷键后期支持配置

### 7. 任务创建表单

按 `n` 后弹出表单，字段顺序：
1. 标题（必填）
2. 截止日期（可选，格式 `YYYY-MM-DD`）
3. 优先级（可选，选择 high/medium/low）
4. 分类（可选，自由输入）
5. 备注（可选，多行文本）

### 8. 任务编辑

按 `e` 进入编辑模式：
- 上下键选择字段
- 回车进入该字段编辑
- Esc 返回任务列表

## 数据存储

### 文件结构

```
.tick/
├── tasks.json      # 所有任务数据
└── config.json     # 配置文件（快捷键配置等）
```

### tasks.json 示例

```json
{
  "tasks": [
    {
      "id": "uuid-1",
      "title": "完成项目报告",
      "status": "pending",
      "priority": "high",
      "due_date": "2026-04-25",
      "category": "工作",
      "note": "需要包含Q1数据",
      "created_at": "2026-04-20T10:30:00",
      "completed_at": null
    }
  ]
}
```

## 交互设计原则

1. **简洁优先** - 避免过多层级，核心操作一键完成
2. **键盘驱动** - 全程键盘操作，无需鼠标
3. **视觉清晰** - 利用颜色区分优先级和状态
4. **轻量快速** - 启动快，操作流畅

## 附录：讨论记录

1. **技术栈选择** - 选择Python + textual，理由是个人使用、开发效率高
2. **同步方案** - 不需要，换电脑时手动迁移JSON文件即可
3. **提醒功能** - 删除，用户会经常查看
4. **搜索功能** - 删除，任务量小不需要
5. **撤销功能** - 删除，保持简洁
6. **已完成任务显示** - 当天显示，第二天后隐藏
7. **任务创建流程** - 弹出表单填写各字段
